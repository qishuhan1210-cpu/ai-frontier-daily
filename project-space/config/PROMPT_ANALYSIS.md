# 提示词文件职责分析

**职责**：定义 LLM 的角色和具体任务

**核心内容**：
- 角色定位：资深 AI 科技媒体编辑
- 三大任务：
  1. 去重评估（8-12条热点收敛）
  2. 摘要强化（6个字段填充）

**位置**：用户消息的**开头部分**

---

- 硬性自检规则（5项）

**位置**：用户消息的**中间部分**

---

### 3. `system.md.j2` - 系统指令
**职责**：LLM 的系统级约束

**核心内容**：
- 身份定位：资深 AI 科技媒体编辑

**位置**：系统消息（system role）

---

### 4. `runtime_injection.md.j2` - 运行时注入
**职责**：动态填充当日数据

**核心内容**：
- 日期、覆盖范围、模块 ID 列表
- 新闻清单 JSON

**位置**：用户消息的**末尾部分**

## 二、合并评估

### 当前拼接顺序（`summarize.py:build_user_prompt`）
```
task + "\n\n---\n\n" + protocol + "\n\n" + injection
```

### 合并建议

| 方案 | 描述 | 优缺点 |
|------|------|--------|
| **保持现状** | 四文件独立 | 清晰分离关注点，便于单独维护 |
| **合并 task+protocol** | 两文件合并为一份完整指令 | 减少文件数量，逻辑更连贯 |
| **完全合并** | 所有内容合并为单一文件 | 简化管理，但可读性下降 |

**推荐方案**：保持现状

**理由**：
1. 四个文件职责清晰，边界明确
2. task 和 protocol 需要频繁修改，独立文件便于版本管理
3. system 和 runtime_injection 相对稳定，且有模板特性（.j2）
